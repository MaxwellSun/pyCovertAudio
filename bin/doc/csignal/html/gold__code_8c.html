Detailed Description

The impementation of Gold Codes. The implementation simply uses two m-sequences of the same degree that meet the "preferred pair" criteria to generate Gold Code sequences.

Gets the next in_number_of_bits from the gold code sequence generator configured in in_gold_code. The generated bits are returned as a byte array in out_code. The size of the byte array is in out_size.

Parameters
in_gold_codeThe generator and initial state of the gold code sequence generator.
in_number_of_code_bitsThe number of code bits to get from the LFSRs.
out_sizeThe size of the byte array containing the code bits (in bytes).
out_codeThe code bits generated
Returns
Returns NO_ERROR upon succesful execution or one of these errors (see cpc_safe_malloc and csignal_get_spreading_code for other possible errors):

CPC_ERROR_CODE_NULL_POINTER If in_gold_code, out_code, or out_size are Null. CPC_ERROR_CODE_INVALID_PARAMETER If in_number_of_bits is zero

csignal_error_code csignal_initialize_gold_code ( UCHAR  in_degree,
UINT32  in_generator_polynomial_1,

UINT32  in_initial_state_2,
gold_code **  out_gold_code 
)

Initializes a gold code generator struct, i.e. two LFSRs. It also ensures that the generator polynomials are valid.

Note
The LSB of both polynomials must be a '1'. Since we are dealing with simple LFSP the g_r term is always assumed to be 1.
Parameters
in_degreeThe degree of the generator polynomials.
in_generator_polynomial_1The generator polynomial for the first LFSR.
in_generator_polynomial_2The generator polynomial for the second LFSR.
in_initial_state_1The initial state of the first LFSR.
in_initial_state_2The initial state of the second LFSR.
out_gold_codeIf no errors are detected a new spreading_code struct will be created and returned. The caller must free this structure.
Returns
Returns NO_ERROR upon succesful execution or one of these errors (see cpc_safe_malloc and csignal_initialize_spreading_code for other possible errors):

CPC_ERROR_CODE_NULL_POINTER If out_gold_code is NULL
