generator_polynomial

The generator polynomial must be a primitive polynomial to ensure an m-sequence is generated. The generator polynomial has a '1' bit set if there is a closed tap in that position in the LFSR. If a 4 stage LFSR was being used with the generator polynomial bit vector [ 1010 0000 0000 0000 0000 0000 0000 0000 ] it would indicate that the 2nd and 4th taps are closed.

Note
The LSB in the polynomial MUST BE '1' (that is the bit in the 32 - degree position). The MSB is implicitly '1'.

state

This is the current state of the LFSR, i.e. the configuration of all the stages. Each bit in the code in the sequence that is generated bit-shifts the state variable 1 position to the right and unneeded bits are masked away.

Note
If the state is initialized to 0, the LFSR will always generate '0' bits.
