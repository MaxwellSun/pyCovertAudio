csignal_error_code csignal_write_FLOAT_wav ( CHAR *  in_file_name,

FLOAT64 **  in_samples 
)

Main entry point to create a WAV file. Will write the data stored in in_samples to a file named in_file_name. The WAV file will have in_number_of_channels channels, be sampled at in_sample_rate and each data stream (one per channel) will have in_number_of_samples. The in_samples array will be a two dimensional array, the first dimension will be the number of channels (should be equal in length to in_number_of_channels) and the second dimension will be samples (should be equal in length to in_number_of_samples).

Note
The WAV format expects float values between -1.0 and +1.0. This function DOES NOT check that the samples in in_samples adhere to this constraint (for reasons of performance). It is up to the caller to ensure this constraint is respected.
Parameters
in_file_nameThe file name to use when creating the WAV file. This function will fail if the file already exists.
in_number_of_channelsThe length of in_samples. There must be a data stream for each channel. Must be smaller than or equal to a FLOAT64.
in_sample_rateThe sample rate that the samples were generated at.
in_number_of_samplesThe number of samples in each channel buffer. Must be smaller than or equal to a UINT32.
in_samplesThe samples array it is an in_number_of_channels X in_samples data buffer.
Returns
Returns NO_ERROR upon succesful execution or one of these errors: 
    CPC_ERROR_CODE_INVALID_PARAMETER  If a file with the name
                                      in_file_name already exists or
                                      could not be created. If the
                                      number of channels or the number
                                      of samples is larger than a UINT16
                                      or UINT32, respectively.
    CSIGNAL_ERROR_CODE_WRITE_ERROR  If data could not be written to
                            the WAV file.
